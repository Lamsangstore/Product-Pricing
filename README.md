# คำนวณต้นทุน & ตั้งราคาสินค้า

เว็บแอปไฟล์เดียวสำหรับคำนวณราคาขายที่ควรตั้ง เพื่อให้ได้กำไรตามที่ต้องการ
รองรับ VAT 7% (แบบผู้จดทะเบียน) และช่องทาง TikTok / Shopee / Lazada / Page365 (แยกค่าอิสระต่อแพลตฟอร์ม)

## วิธีเปิดใช้งาน

- **ง่ายสุด:** ดับเบิลคลิกไฟล์ `index.html` เปิดในเบราว์เซอร์ได้เลย (ใช้ได้ทั้งมือถือ/คอม ไม่ต้องต่อเน็ต)
- **แบบเซิร์ฟเวอร์ (แนะนำถ้าจะใช้ Firebase):** `node server.js` แล้วเปิด http://localhost:4173

> การซิงก์ Firebase ต้องเปิดผ่านเซิร์ฟเวอร์ (localhost หรือโฮสต์จริง) — ถ้าเปิดแบบดับเบิลคลิก (`file://`) ตัวคำนวณใช้ได้ปกติ แต่การซิงก์คลาวด์อาจไม่ทำงาน

## ความสามารถ

- ใส่ต้นทุน + กำไรที่ต้องการ → คำนวณ **ราคาขายที่ควรตั้ง** และ **ราคาเผื่อต่อรอง** ให้อัตโนมัติ
- แยกสัดส่วนราคาขายเป็นแถบสี (ต้นทุน / การตลาด / ค่าธรรมเนียม / VAT / กำไร)
- **จุดคุ้มทุน**: ต้องขายกี่ชิ้น/เดือนถึงไม่ขาดทุน + ประมาณการกำไรสุทธิตามเป้ายอดขาย
- เช็ก **กำไรจริง** เมื่อตั้งราคาอื่น
- กรอกค่าการตลาดด้วย % หรือ ROAS ก็ได้ (คำนวณสลับกันให้)
- VAT แบบหักภาษีซื้อ (Input VAT) เปิด/ปิดได้
- บันทึกสินค้าหลายรายการ (เก็บใน localStorage) + Export CSV
- **ซิงก์ขึ้นคลาวด์ด้วย Firebase** (Anonymous auth + Firestore) — เก็บเป็นแบ็กอัพ ผูกกับเครื่อง/เบราว์เซอร์นี้

## ตั้งค่า Firebase (สำหรับซิงก์คลาวด์)

1. ไปที่ [Firebase Console](https://console.firebase.google.com) → เปิดโปรเจกต์ของคุณ
2. **เพิ่ม Web app** (ถ้ายังไม่มี): ⚙️ Project settings → General → Your apps → `</>`
3. ก๊อป `firebaseConfig` มาวางใน **`firebase-config.js`** (แทนค่า `PASTE_...`)
4. เปิด **Anonymous auth**: Authentication → Sign-in method → Anonymous → Enable
5. สร้าง **Firestore**: Firestore Database → Create database
6. ตั้ง **Security Rules** (Firestore → Rules) ให้แต่ละคนเห็นเฉพาะข้อมูลตัวเอง:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid}/products/{doc} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }
  }
}
```

7. เปิดแอปผ่าน `node server.js` → ป้ายมุมขวาบนจะขึ้น **"● ซิงก์แล้ว"** เมื่อเชื่อมต่อสำเร็จ

> ข้อมูลเก็บที่ `users/{uid}/products` โดย `uid` เป็นบัญชี Anonymous ที่ผูกกับเบราว์เซอร์นี้ — ล้าง site data หรือเปลี่ยนเครื่อง = คนละ uid (ข้อมูลไม่ข้ามเครื่องอัตโนมัติ) ถ้าต้องการซิงก์ข้ามเครื่องจริง ให้เปลี่ยนไปใช้ Google Sign-in ภายหลัง

## สูตรหลัก

```
ราคาขาย = (ต้นทุนบาท + กำไรที่ต้องการ − เครดิต VAT ซื้อ)
          ÷ (1 − %การตลาด − %แพลตฟอร์ม − %ชำระเงิน − %VAT)

จุดคุ้มทุน (ชิ้น/เดือน) = ต้นทุนคงที่ต่อเดือน ÷ กำไรต่อชิ้น
```

> หมายเหตุ: ค่าธรรมเนียมเริ่มต้นของแต่ละแพลตฟอร์มเป็นค่าประมาณ ควรปรับตามอัตราจริงของร้าน
